## 2.X Approaches to Graph-Based Retrieval

Recent literature highlights several methodologies for extracting subgraphs and calculating retrieval scores within GraphRAG systems. Each approach solves the challenge of retrieving structured topological information alongside textual data differently.

Microsoft's GraphRAG focuses heavily on global query comprehension (edge2024local). Their approach uses Leiden community detection to partition the knowledge graph into hierarchical communities. Instead of retrieving isolated nodes, the system extracts entire community subgraphs. It then generates pre-computed summaries for each community. During retrieval, the system scores these community summaries against the user query rather than scoring individual edges or nodes. This allows the model to answer broad, corpus-wide questions by aggregating information from the most relevant high-level communities.

The Neo4j ecosystem provides a more traditional but highly optimized approach to GraphRAG (neo4j2024essential). Neo4j typically relies on hybrid retrieval mechanisms. It combines vector similarity search on node properties with Cypher-based graph traversals. Subgraph extraction in Neo4j usually involves finding starting nodes via vector search and then expanding outward by a predefined number of hops. Retrieval scoring is often a weighted combination of the initial vector similarity score and graph-based metrics like PageRank or path distance. This method excels at multi-hop reasoning where explicit relationships define the context.

SubgraphRAG (subgraphrag2025) takes a fundamentally different path by treating subgraph extraction as a single-step machine learning problem. Rather than relying on complex, multi-round graph traversals or expensive Large Language Model generation during the retrieval phase, SubgraphRAG employs a lightweight Multi-Layer Perceptron. This model directly scores candidate graph structures. It evaluates directional structural distances to find the optimal subgraph size that matches both the query requirements and the downstream model's context limits. This single-step retrieval drastically reduces latency compared to iterative extraction methods.

GRAG (grag2025) addresses the challenge of retrieving textual subgraphs through a divide-and-conquer strategy. It focuses specifically on environments where documents themselves form a network, such as citation graphs or social media. GRAG identifies optimal subgraph structures in linear time by recursively dividing the graph space. For retrieval scoring, GRAG incorporates two complementary views: a text view and a graph view. It explicitly scores the joint textual and topological relevance of the subgraph, ensuring the language model receives both the content of the nodes and the structural context of their connections.

In summary, Microsoft uses hierarchical community summaries for global understanding. Neo4j relies on vector search combined with explicit hop traversals. SubgraphRAG optimizes for speed using a single-step neural network to score subgraphs. GRAG uses a divide-and-conquer algorithm to balance text and topology in networked documents.
