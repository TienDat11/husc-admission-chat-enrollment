# PaddedRAG Architecture Audit Report

## [ARCH] Current PaddedRAG Architecture

### Retriever
- **Primary**: HybridRetriever (services/retriever.py:24) - Dense + Sparse + Reranking
- **Components**:
  - Dense Search: Vector similarity using BGE-M3 (1024-dim)
  - Sparse Search: BM25 with Vietnamese tokenization
  - Fusion: Reciprocal Rank Fusion (RRF) with weights (dense=0.6, sparse=0.4)
  - Reranker: BAAI/bge-reranker-base cross-encoder

### Reranker
- **Model**: BAAI/bge-reranker-base (config/settings.py:48)
- **Usage**: Applied to top 50 candidates after RRF fusion

### LLM
- **Primary**: GLM-4.5 (Z.AI) via ZaiClient (services/llm_generator.py:44)
- **Fallback**: Groq Llama 3.1 70B (services/llm_generator.py:57)
- **Generation**: Anti-redundancy optimized prompt (50-120 words target)

## [NEO4J] Neo4j Connection
- **Status**: NO Neo4j integration found in current architecture
- **Note**: Only reference found in download_pdf.py:4 (downloads Neo4j GraphRAG PDF)
- **Authentication**: Not applicable

## [DATA] Chunked JSONL Loading & Indexing

### File Loading
- **Pattern**: chunked_*.jsonl files (scripts/ingest_all_chunks.py:67)
- **Location**: data/chunked/ directory
- **Processing**: All JSONL files processed except "enhanced" files

### Indexing Pipeline
1. **Chunking**: Adaptive chunker with profiles (auto, faq, policy)
2. **Embedding**: BGE-M3 model (1024-dim) with normalization
3. **Storage**: Qdrant collection with proper payload structure
4. **Payload Structure**:
   - text (MANDATORY)
   - source
   - chunk_id
   - metadata (nested)
   - faq_type (root level)
